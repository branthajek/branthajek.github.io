Professional Work Portfolio

Applyweb forms

These forms are typically used by graduate schools (i.e., Stanford, Cornell, University of Michigan) with a high degree of complexity. We develop highly customizable forms to fit the client's business needs which include optional features such as transcript verification, asynchronous letter of recommendation requests, video interview questions, just to name a few. In order to access or view these forms you need an Applyweb account - here is an example of a form I built recently.

Applyweb Self Service Dashboard

Applyweb forms now offer self-service features, allowing our clients to make certain edits to their own forms and publish them without having to submit requests to the development team. I was apart of the team that developed the UI for the dashboard in which the client can view, edit, and publish their forms. It was developed in Angular, using NgRx for state management and our in-house component library. This is a strictly client-facing product.

Applyweb Letter of Recommendation Dashboard

This dashboard is where recommenders, usually college professors, can manage their requests from current or former students, colleagues, or employees and begin filling out a separate evaluation form (which we also develop and maintain). This is another example of a UI built in Angular that I have contributed to. This dashboard is only accessible if you have received an email with a link to login.

CollegeNET Pattern Library

This is a strictly internal project which contains all of the components and features our Angular projects pull in and use. Occasionally, when working on other projects, I have had to make updates to this one so that a certain component or feature may meet the needs outlined by the design team. This is a large, complex project, architected to be used by various applications accross multiple departments.
